Project: eRabb.it

A terminal-esque web profile

Stack

  • - CSS: Tailwind
  • - DevOps: Vite
  • - JS: Turbo + Stimulus

Objective

To use Turbo Frames to create a UX that behaved like a terminal with as little custom JS as practical, and with no backend what-so-ever.

Benefits

  • - Generates static HTML for consistently fast page loads
  • - Sends HTML-over-the-wire for fast, dynamic page updates
  • - Adds very little custom JS so the application overhead remains low
  • - Doesn't utilize a backend server or database so the entire application can be hosted for free on GitHub pages

Drawbacks

  • - With no backend, forces a trade-off between DRY-templating and superfluous requests. Opted for the latter as these typically produce 304s from the browser, and/or are fast and free. These templates can be found in the /public/application directory.
